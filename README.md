# PI-Map Paper Reproduction

Code and compact data for reproducing the figures and tables for:

> Efficient Low-Dimensional Spectral Rendering with Illuminant-Specific Linear Maps

The repository is intentionally limited to the paper-reproduction path. It excludes exploratory notebooks, local virtual environments, generated bytecode, and machine-specific files.

## Environment

Tested with Python 3.14.2, NumPy 2.3.4, Matplotlib 3.10.7, and LuxPy 1.12.0.

```bash
python3.14 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

If `python3.14` is not available, use a recent Python 3 release supported by the pinned dependencies.

Conda users can instead run:

```bash
conda env create -f environment.yml
conda activate pimap-repro
```

## Reproduction Commands

Run all commands from the repository root.

Fast reproduction from the bundled compact NPZ:

```bash
python main.py --mode npz all
```

Full recomputation from LuxPy spectral data:

```bash
python main.py --mode full all
```

Rebuild the compact NPZ from scratch, then regenerate figures and tables from that new cache:

```bash
bash scripts/rebuild_full.sh
```

Quick smoke test:

```bash
bash scripts/smoke_test.sh
```

Useful partial commands:

```bash
python main.py inventory
python main.py --mode npz figures
python main.py --mode npz tables
python main.py --mode npz basis
python main.py --mode full pack-npz
```

For a short full-path check without the paper-scale sampling/training budget:

```bash
python main.py --mode full --full-basis-epochs 1 --full-max-combos 10 --full-cov-paths 50 tables
```

The default full recomputation uses the numerical settings in `code/presets.py`, including seed 0, 100000 covariance paths, 4000 sampled reflectance-path combinations per condition, and 25000 Gaussian-basis optimization epochs for each K in {3, 4, 5, 6}.

Use `--mode npz` for quick verification. The full path reruns sampling,
model fitting, evaluation, and Gaussian-basis optimization; expect it to take
substantially longer than the NPZ path and to depend on CPU/GPU availability.

## Outputs

Generated artifacts are written under `results/`:

- `results/Figures/`: manuscript figure PDFs and selected PNG mosaics
- `results/tables/`: manuscript CSV and TeX tables

The default figure run includes the three-dataset reflectance covariance panel,
IQR error panels, learned basis panel, and Macbeth multi-illuminant comparison
mosaic.

## Data Provenance

Full mode loads standard CIE illuminants, CMFs, CIE 224:2017 Rf reflectances, Macbeth reflectances, and Munsell reflectances through LuxPy. The bundled `data/cache/paper_data_de2000_jab.npz` is a compact paper-data artifact produced from the same public code path and is used only by `--mode npz`.

Data-dependent methods are fit on the CIE 224 Rf pool. The fixed-basis and learned-vs-Wandell percentile tables are evaluated on the held-out Macbeth + Munsell reflectance pools.

## Layout

```text
main.py                  paper figure/table command line entry point
code/                    focused reproduction modules
data/cache/              compact NPZ for fast reproduction
results/                 generated outputs
scripts/                 smoke-test and full-rebuild helpers
```
